# 모니터링 시스템 센서 매핑 및 식별 개선 계획

현재 모니터링 시스템의 센서 데이터 매핑 오류(데이터 중첩)와 식별자 미흡 문제를 해결하기 위한 계획입니다. `sensor_id_system.md` 정의에 따라 128개 센서를 정확히 구분하고 명칭을 직관적으로 수정합니다.

## 문제 현황
1. **데이터 중첩**: 현재 `(모듈 * 16) + 유닛번호` 방식으로 인덱싱을 하고 있어, 동일 모듈 내 다른 타입의 센서(예: Temperature 0번과 Force 0번)가 같은 셀을 점유하고 있습니다.
2. **식별자 모호성**: 대시보드 타일이 `S-00`과 같은 16진수 번호로만 표시되어 어떤 센서인지 알기 어렵습니다.

## 주요 수정 사항

### 1. 센서 메타데이터 및 매핑 로직 구축 (`monitoring/app.js`)
- `sensor_id_system.md`의 내용을 기반으로 전체 센서 구성표(`SENSOR_LAYOUT`)를 작성합니다.
- 128개 전체 센서 유닛에 대해 `(모듈, 타입, 유닛번호)`를 키로 하고 `글로벌 인덱스(0-127)`를 값으로 하는 룩업 테이블을 생성합니다.
- 데이터 수신 시 이 테이블을 사용하여 정확한 타일 위치를 찾습니다.

### 2. UI 레이아웃 및 라벨링 개선 (`monitoring/app.js` & `index.html`)
- **타일 라벨**: `S-00` 대신 `모듈 명칭 - 센서 타입 - 번호` (예: `Bond Head-Temp-1`) 형식을 적용합니다.
- **상세 패널**: 우측 상세 분석 창의 헤더에도 동일한 명칭 체계를 적용합니다.
- **가독성**: 폰트 크기 및 레이아웃을 조정하여 긴 명칭도 잘 보이도록 최적화합니다.

### 3. 서버 전송 데이터 정합성 확인
- 서버가 보내는 `m(모듈)`, `t(타입)`, `i(유닛번호)` 값이 위 매핑 체계와 일치하는지 최종 확인합니다.

## 작업 순서
1. `app.js`에 `sensor_id_system.md` 기반 정적 매핑 데이터 추가
2. `initSensorGrid()` 함수를 수정하여 정적 매핑 순서대로 타일 생성 및 라벨링
3. `processIncomingData()`의 인덱스 계산 로직을 룩업 테이블 참조 방식으로 변경
4. 상세 패널(`renderDetailPanel`)의 명칭 출력 로직 수정
5. 실제 구동 테스트 및 데이터 정합성 확인

## 기대 결과
- 128개 센서가 중첩 없이 각각 독립된 셀에 정확히 표시됩니다.
- 모니터링 화면만으로도 어떤 모듈의 어떤 센서인지 즉시 파악 가능합니다.
